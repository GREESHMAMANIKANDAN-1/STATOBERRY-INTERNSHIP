Principal Component Analysis

1Statoberry LLP, 2Department of Agricultural Statistics, Kerala Agricultural University

ABSTRACT

Principal Component Analysis (PCA) is a powerful multivariate statistical technique used to reduce the dimensionality of a dataset while retaining as much of the original variation as possible. PCA transforms a set of correlated variables into a smaller number of uncorrelated components called principal components, each capturing a decreasing proportion of the total variance in the data. In RAISINS you can perform PCA very easily without writing a single line of code. This tutorial will guide you through how to perform PCA in RAISINS and interpret the results effectively. In addition, you will get publication-ready tables, scree plots, biplots, and index scores. You can also explore cluster structure in your data using HCPC (Hierarchical Clustering on Principal Components) and examine variable relationships through Correlation Analysis.

Hover or click each point to see more information.

Introduction Principal Component Analysis
    The origins of PCA trace back to the early twentieth century. In 1901, mathematician and statistician Karl Pearson introduced the concept of fitting lines and planes to data clouds, laying the groundwork for what would later be formalized as Principal Component Analysis. Pearson described the technique as a way to find the axis of closest fit through a swarm of points in multidimensional space, a deceptively simple idea that would prove transformative for statistics, genetics, ecology, and social science alike. Harold Hotelling independently reformulated and extended PCA in 1933, introducing the modern computational framework and demonstrating its practical utility for analyzing correlated measurements. Since then, PCA has become one of the most widely used tools in multivariate analysis, employed across disciplines ranging from agricultural research and plant breeding to genomics, remote sensing, finance, and machine learning. Its strength lies in its ability to summarize complex, high-dimensional data into interpretable components, revealing hidden structure and enabling efficient comparison of observations.

2 Principal Component Analysis (PCA)

Principal Component Analysis (PCA) is a multivariate statistical technique designed to reduce the dimensionality of a dataset consisting of a large number of interrelated variables while retaining as much of the variation present in the dataset as possible. It achieves this by transforming the original correlated variables into a new set of uncorrelated variables called principal components (PCs), which are ordered so that the first few retain most of the variation present in all of the original variables. PCA is widely applied in agricultural research, plant breeding, ecology, genetics, and social sciences wherever there is a need to summarize multiple measurements recorded on the same set of observations. It is particularly valuable when variables are correlated, as correlation implies redundancy and PCA removes that redundancy by rotating the data to a new coordinate system. It is important to note that PCA is an exploratory technique and not an inferential method — it reveals structure in the data but does not test hypotheses about population parameters.

TipPrincipal Component Analysis (PCA) is a dimensionality reduction technique that transforms a set of correlated variables into a smaller number of uncorrelated principal components, each accounting for a decreasing proportion of the total variance in the data.

3 A working example

To make things simple and interesting, we will explain PCA step by step using a hypothetical example, so you can clearly see how it works and why it matters. Let us assume an experiment conducted to evaluate twenty rice genotypes across a set of agro-morphological and quality traits. The genotypes are named G1 through G20. Observations were recorded for five variables: GY (Grain Yield in g/plant), PH (Plant Height in cm), DTF (Days to Flowering), TGW (Thousand Grain Weight in g), and CL (Chalkiness percentage). Our aim is to use PCA to uncover the underlying structure of variation among these genotypes, identify traits that contribute most to differentiation, and derive a composite index score to rank genotypes. 6 Analyse Results

Once your dataset is uploaded and Run Analysis is clicked, RAISINS will perform PCA and display a comprehensive set of outputs. The analysis begins with an Eigenvalue Table, which summarizes how much variance each principal component explains (see Figure 5). Eigenvalues represent the amount of variance associated with each principal component. Components with eigenvalues greater than 1.0 (the Kaiser criterion) are generally considered important for retention and further interpretation.

Table 1: Eigenvalues and Variance Explained

Understanding the Eigenvalue Table

The Eigenvalue Table produced by RAISINS contains the following columns:

Principal Component (PC): The sequential label for each derived component (PC1, PC2, PC3, …). They are ordered from the component capturing the most variance to the least.

Eigenvalue: The variance associated with each principal component. An eigenvalue of 1 means the component explains as much variance as one standardized original variable. Components with eigenvalues below 1 are generally discarded.

Proportion of Variance (%): The percentage of total variance in the dataset explained by each component. PC1 always has the highest proportion.

Cumulative Variance (%): The running total of variance explained as successive principal components are included. Analysts often retain components until the cumulative variance reaches 70–80%, as this threshold indicates that the retained components adequately summarize the data.

Significance is indicated for the number of PCs recommended for retention based on the Kaiser criterion (eigenvalue > 1) and the scree plot elbow method.

6.1 Interpretation from Figure 5

In our hypothetical rice genotype example, PC1 accounts for approximately 42.5% of the total variance, while PC2 accounts for about 26.8% of the variance. Together, PC1 and PC2 explain approximately 69.3% of the total variance in the dataset. Both PC1 and PC2 have eigenvalues greater than 1 and are therefore retained for further analysis. Since PC1 alone explains more than 40% of the total variance, a PC1-based selection index is a strong and defensible choice. Since the combined cumulative variance exceeds 60%, an index based on both PC1 and PC2 is also appropriate. PC3 and beyond have progressively smaller eigenvalues and explain relatively minor proportions of variance, making them less informative for interpretation in this example.

Table 2: Component Loadings (Variable Contributions to Each PC)

Overview of Loadings Table and Interpretation

Loadings: The correlation between each original variable and each principal component. Loadings range from −1 to +1. A large positive loading (e.g., > 0.5) means the variable contributes strongly and positively to the component. A large negative loading means the variable contributes strongly but inversely. A loading near zero means the variable has little relationship with that component.

PC1 Loadings: Variables with high positive loadings in PC1 tend to increase together and form the dominant axis of variation. Genotypes with high PC1 scores perform well on these traits.

PC2 Loadings: Variables that have small loadings in PC1 but high loadings in PC2 represent a secondary axis of variation, capturing patterns not already explained by PC1.

Communality: The proportion of each variable’s variance that is explained by the retained principal components. High communality (close to 1) indicates that the retained PCs capture most of the information in that variable.

Interpretation tip: When selecting genotypes based on PCA, identify which traits you wish to improve, look at the sign of their loadings in the relevant PC, and choose genotypes with index scores in the corresponding direction.

6.2 Interpretation from Figure 6

In our example, GY and TGW show strong positive loadings on PC1 (approximately +0.72 and +0.68 respectively), indicating that high-yielding genotypes with heavy grains score high on PC1. DTF and CL show negative loadings on PC1 (approximately −0.61 and −0.58), suggesting that late-flowering and chalky genotypes score lower on PC1 — which is desirable for CL reduction. PH shows moderate positive loadings across both PC1 and PC2, indicating it contributes to variation captured by both components. Genotypes positioned in the positive direction of PC1 in the biplot therefore represent high-yielding, heavy-grained, early-flowering, and low-chalkiness accessions, making them the primary candidates for selection.

TipThe sign of a loading indicates direction — if GY has a positive loading on PC1, genotypes with high PC1 scores are expected to have high grain yield. Selecting genotypes based on the loading signs helps translate PCA results into actionable breeding decisions.

Treatments sharing similar PC score patterns are positioned close together in the biplot, while those that are very different are placed far apart. The biplot thus provides a powerful visual summary of both similarity among observations and the contributions of variables to the principal components.

7 Multiple Comparison Tests

PCA itself is not a hypothesis-testing procedure and does not generate p-values for comparing groups directly. However, RAISINS integrates PCA with subsequent analyses that do involve group comparisons. Once principal component scores are computed, users may proceed to HCPC (see Figure 28) to identify clusters of similar observations, or return to ANOVA-based modules to compare group means on individual components. In the context of PCA-based selection indices, the ranking of genotypes by their scaled index score serves as the primary criterion for ordering and selecting superior entries. The Critical Difference (CD) and post-hoc grouping letters that appear in complementary ANOVA results help establish which genotype groups are statistically distinct on individual traits, providing inferential support for selections guided by PCA.

When to combine PCA with post-hoc tests

PCA is exploratory — it summarizes multivariate data but does not tell you whether the differences among groups are statistically significant. For rigorous inference, complement PCA with:

MANOVA: Tests whether groups differ significantly across all variables simultaneously. If MANOVA is significant, PCA helps visualize where and how the groups differ.

Individual ANOVA with post-hoc tests (LSD, Tukey’s HSD, DMRT): Tests significance on each variable separately. LSD (Least Significant Difference) is the most liberal test, suitable when few comparisons are made. Tukey’s HSD controls the family-wise error rate and is recommended for all pairwise comparisons. DMRT (Duncan’s Multiple Range Test) is intermediate in stringency and widely used in agricultural research.

The combination of PCA (structure discovery) and ANOVA post-hoc tests (significance testing) gives the most complete picture of multivariate treatment differences.

8 Summary Stats

Before interpreting PCA results, it is essential to examine the descriptive statistics of each variable. In RAISINS, click on the Summary Stats tab within the PCA module to access a table that shows Mean, Standard Deviation (SD), Standard Error (SE), Minimum, Maximum, Coefficient of Variation (CV%), Skewness, and Kurtosis for each variable across all observations, as shown in Figure 8.

Table parameters

Mean: The arithmetic average of all observations for the variable. It provides a central reference point.

SD (Standard Deviation): A measure of dispersion around the mean. Higher SD indicates greater variability across genotypes or observations for that trait.

SE (Standard Error): Estimates how far the observed mean is likely to deviate from the true population mean. Calculated as \[SE = \frac{SD}{\sqrt{n}}\], where n is the number of observations.

Min / Max: The lowest and highest recorded values for the variable, providing the range of observed performance.

CV (Coefficient of Variation): Expresses the standard deviation as a percentage of the mean: \[CV = \frac{SD}{\text{Mean}} \times 100\]. A low CV indicates high consistency; a high CV suggests large variability, which may mean the variable has discriminating power useful for PCA.

Skewness: Measures the asymmetry of the distribution. A value close to zero indicates symmetry. Positive values indicate a longer right tail (a few high-performing outliers); negative values indicate a longer left tail.

Kurtosis: Measures the “tailedness” of the distribution relative to a normal distribution (kurtosis = 3). Values above 3 indicate a sharper peak with heavier tails; values below 3 indicate a flatter distribution.

8.1 Interpretations from Figure 8

In our rice genotype example, GY shows a mean of 28.4 g/plant with a CV of 18.2%, indicating substantial variation among genotypes that is well-suited for PCA-based discrimination. TGW shows a mean of 22.1 g with a CV of 12.5%, while DTF has the lowest CV (8.3%), suggesting less variation in days to flowering across the set. CL exhibits moderate skewness (0.72), reflecting a few genotypes with notably high chalkiness values that may be driving separation in a specific component. PH shows near-symmetric distribution and moderate variability. These patterns confirm that the five variables capture distinct and complementary aspects of genotype performance, making the dataset appropriate for PCA.

9 Individual ANOVA

To examine the significance of variation for each individual variable before or alongside PCA, click on Individual ANOVA in the Analysis tab. This provides a separate ANOVA table for each variable, allowing you to determine which traits show statistically significant differences among genotypes or treatments, as shown in Figure 9.

Table 4: ANOVA table for Grain Yield (GY)

Table parameters

Critical Difference (CD)

The minimum difference required between any two treatment means to declare them significantly different. At the 5% level, you are 95% confident in the declared difference; at the 1% level, 99% confident.

Coefficient of Variation (CV (%))

A relative measure of experimental precision: \[CV = \frac{SD}{\text{Mean}} \times 100\] A low CV (below 15% for field experiments) indicates that experimental error is small relative to the treatment means, lending credibility to the ANOVA findings.

Mean Square Error (MSE)

The residual mean square from the ANOVA table. It represents unexplained variance (noise) within groups and is used to compute SE and CD values.

Standard Error of Mean SE(m)

Measures how much a treatment mean is expected to vary from the true population mean: \[SE(m) = \sqrt{MSE / r}\] where r is the number of replications or the number of observations per group.

Standard Error of Difference SE(d)

The standard error associated with the difference between two treatment means: \[SE(d) = \sqrt{2 \times MSE / r}\] It is the direct basis for computing the CD value.

9.1 Interpretation from Figure 9

The individual ANOVA for GY shows a genotype mean square of 182.4, considerably larger than the error mean square of 31.6, yielding an F-value of 5.77 significant at the 1% level (p < 0.01). This confirms that grain yield varies significantly among the twenty rice genotypes, validating the use of GY as a discriminating variable in PCA. The CV of 9.8% is satisfactory for a field-based evaluation. The CD at the 5% level is 8.34 g/plant, and at the 1% level is 11.27 g/plant, values that can be used to identify pairs of genotypes with significantly different yields when complementing PCA results with direct pairwise comparisons.

10.1 Customizing plots

Click the gear icon on any plot to access the customization panel, as shown in Figure 11. You can modify axis labels, title text, color palettes, point size, and font size to match your publication requirements.

A box plot compares the distribution of each variable across all observations. Each colored box represents a variable and shows key statistics: the median (middle line), interquartile range (the box), minimum and maximum (whiskers), and outliers (individual points). This plot is useful for identifying variables with wide spread (high discriminating potential) versus those with narrow ranges (low variation, less suitable for PCA).

A violin plot combines a density plot and box plot to show the full distribution of each variable. Wider sections of the violin indicate where values are most concentrated. Variables with multi-modal distributions may reflect natural groupings among observations — a pattern that PCA will reveal as distinct clusters in score space.

A bar plot displays the mean value of each variable (or of each observation for a selected variable) with error bars representing variability. It provides a straightforward visual summary of average performance, allowing quick identification of variables with the greatest overall mean differences across genotypes or treatments.

A mean value plot displays observation means (e.g., genotype means) for a selected variable as colored dots with horizontal error bars. Observations are listed vertically, allowing comparison of performance across the full set. This plot complements PCA by confirming the ranking of individual observations on specific traits.

A connected line plot links mean values of successive observations with lines, highlighting trends and ranking patterns across genotypes or treatments. It is particularly useful for visualizing how a set of observations ranks on a specific trait and for spotting unusual performers at either extreme — observations that may represent outlier genotypes or data entry errors.

11 Advanced Plots

RAISINS also provides Advanced Plots which go beyond basic charts to offer deeper insight into data distributions, multivariate relationships, and deviations from normality (see Figure 17). These plots are particularly valuable in the context of PCA, where understanding the structure of the raw data helps interpret the principal components more accurately.

SUMMARY PLOT

A summary plot is a visual representation that shows how data values are distributed across all variables simultaneously. It helps quickly identify which variables exhibit the widest spread, which are most symmetric, and which may contain extreme values — information that directly affects how PCA distributes variance across components.

ADVANCED RAINCLOUD PLOT

The advanced raincloud plot combines a density curve (the cloud), a boxplot, and individual observation points (the rain) to present the complete distributional picture for a selected variable. It reveals whether the data is unimodal or multimodal, whether outliers are present, and how compact the central mass of observations is — all of which influence PCA output.

QQ PLOT

The Q-Q (quantile–quantile) plot assesses whether a variable follows a normal distribution by plotting observed quantiles against theoretical quantiles from a standard normal distribution. Points lying close to the diagonal reference line indicate approximate normality. PCA assumes multivariate normality for inferential extensions such as MANOVA; the QQ plot helps verify this assumption for each variable.

RAIN CLOUD PLOT

The raincloud plot shows the distribution of a selected variable across all observations. The half-violin (cloud) represents the data density, the embedded boxplot summarizes the median and interquartile range, and the jittered dots (rain) display individual observations. This plot is useful for confirming that a variable has sufficient variance to contribute meaningfully to a principal component.

DISTRIBUTION PLOT

The distribution plot shows how the observed values are spread for each variable in the dataset. It provides a visual comparison of the pattern, spread, and variability of measurements, helping to understand which variables carry the most discriminating information — an important consideration before interpreting PCA loadings.

12 Analyse Result

The Analyse Result tab in RAISINS provides the primary numerical outputs of PCA, including the eigenvalue table, variance explained, and component loadings for all variables. This is the central output panel from which all further interpretation begins (see Figure 23). After clicking Run Analysis, this tab is automatically populated. Users can examine how many principal components to retain, which variables drive each component, and how much of the total variability in the data is accounted for by the retained components.

A scree plot is automatically generated alongside the eigenvalue table. The scree plot shows eigenvalues plotted against the component number — a sharp “elbow” in the curve indicates the point beyond which additional components contribute diminishing returns. Retaining components to the left of the elbow is a common rule of thumb that complements the Kaiser criterion (eigenvalue > 1). Together, these two criteria guide the selection of the optimal number of components for interpretation and index construction.

13 Index

The Index tab in RAISINS provides a powerful tool for translating PCA results into a practical ranking of observations (see Figure 25). A PCA-based selection index is constructed by combining the scores of the retained principal components weighted by their respective eigenvalues (proportion of variance explained). This composite index summarizes the overall multivariate performance of each genotype or treatment into a single numerical score, enabling straightforward ranking and selection.

In RAISINS, the index score is further converted to a Scaled Index Score ranging from 0 to 1, making it easier to interpret and compare across datasets and experiments. This standardization ensures that the index is dimensionless and robust to differences in unit of measurement across variables. To refine selection, use the Select cutoff for Scaled Index Score feature as shown in Figure 26. You can specify the cutoff percentage (default is 75%) to identify observations above or below a threshold. By toggling the up-arrow and down-arrow buttons, you can choose the top or bottom percentile of observations based on your breeding or selection objective. Observations meeting the criterion are highlighted in yellow in the ranking table below.

In our rice genotype example, the scaled index based on PC1 and PC2 (cumulative variance 69.3%) ranks G7 at the top (scaled index = 0.91), followed by G3 (0.87) and G15 (0.84). These genotypes exhibit superior combinations of high GY, high TGW, early flowering, and low chalkiness — traits that loaded positively on the dominant components. Applying a 75% cutoff identifies the top five genotypes as elite candidates for further evaluation or advancement in the breeding pipeline.

TipThe direction of the index score depends on how variables are loaded. Always check the loadings table before interpreting index scores — if an undesirable trait (e.g., chalkiness) has a positive loading on PC1, high PC1 scores may not always indicate superior performance for that trait.

14 HCPC — Hierarchical Clustering on Principal Components

HCPC (Hierarchical Clustering on Principal Components) is an advanced clustering technique that combines the dimensionality reduction power of PCA with the grouping capability of hierarchical cluster analysis. In RAISINS, the HCPC tab performs this analysis automatically after PCA, enabling users to identify natural groupings (clusters) among observations in the reduced PC space (see Figure 28).

The clustering proceeds in two steps. First, PCA is performed to obtain component scores for each observation. Second, Ward’s hierarchical clustering algorithm is applied to the PC scores (typically the retained components) to group observations into clusters that are as internally similar and as externally different from one another as possible. The number of clusters is determined by examining the dendrogram (tree diagram) produced during hierarchical clustering — a horizontal cut at the appropriate height divides observations into the chosen number of clusters. RAISINS automatically suggests an optimal number of clusters based on the within-cluster inertia curve and Calinski-Harabasz index, though users may adjust the number of clusters manually.

The cluster map produced by HCPC (see Figure 30) overlays the identified clusters on the PCA biplot, color-coding each observation by its cluster membership. This visualization reveals which observations are most similar across all measured variables simultaneously and helps identify “diversity groups” in a genotype panel — a critical step in designing crosses or selecting contrasting parents in plant breeding. Cluster membership is also reported in tabular form so that each genotype can be assigned to its group.

In our rice example, HCPC identifies three distinct clusters. Cluster 1 (n = 7 genotypes) is characterized by high GY and TGW and low CL — the most desirable cluster. Cluster 2 (n = 8 genotypes) contains intermediate performers with moderate values across all traits. Cluster 3 (n = 5 genotypes) includes late-flowering genotypes with high PH and high CL, representing the least desirable combination for most breeding objectives. This three-cluster solution is supported by the within-cluster inertia curve and aligns with the visual separation observed in the biplot.

Understanding the HCPC outputs

Dendrogram: A tree diagram that displays the hierarchical merging of observations into clusters. The height of each merge reflects the dissimilarity at which two groups were joined. A horizontal line drawn across the dendrogram at a chosen height determines the number of clusters.

Cluster Map: The PCA biplot with observations color-coded by cluster membership. It provides a simultaneous view of how clusters are distributed across the principal component space and how they differ with respect to the original variables.

Cluster Description Table: A table produced by RAISINS showing the mean value of each variable within each cluster and comparing it to the overall mean. Variables with large positive or negative deviations from the overall mean are the “characterizing variables” for that cluster.

Inertia Curve: A plot of within-cluster inertia against the number of clusters. A distinct bend (elbow) in the curve indicates the optimal number of clusters — adding more clusters beyond the elbow produces diminishing improvements in within-cluster homogeneity.

15 Correlation Analysis

The Correlation Analysis tab in RAISINS computes and displays pairwise Pearson correlation coefficients among all selected variables (see Figure 31). Understanding the correlation structure of the data is a crucial prerequisite for PCA, since PCA is most effective when variables are moderately to strongly correlated — correlation implies redundancy, and PCA transforms correlated variables into uncorrelated components to remove this redundancy.

The correlation matrix displays the Pearson r value for each pair of variables, along with significance indicators. Significance is indicated by an asterisk (*) for the 5% level and two asterisks (**) for the 1% level. A correlation heatmap is also displayed, in which cells are color-coded from deep blue (strong negative correlation, r close to −1) through white (no correlation, r ≈ 0) to deep red (strong positive correlation, r close to +1). This heatmap allows rapid visual identification of variable clusters — groups of variables that are highly intercorrelated and therefore likely to load together on the same principal component.

Interpreting the correlation matrix for PCA

Pearson Correlation Coefficient (r): Measures the strength and direction of the linear relationship between two variables. Values range from −1 to +1. r = +1 indicates a perfect positive linear relationship; r = −1 indicates a perfect negative relationship; r = 0 indicates no linear relationship.

Significance of correlation: Even moderate correlations can be statistically significant with large sample sizes. Conversely, strong correlations may not reach significance with very small datasets. Always examine both the r value and its p-value together.

Implications for PCA:

— If all variables are highly positively correlated, PC1 will capture most of the variance and will be interpretable as a general “size” or “performance” factor.

— If variables form two or more groups of correlated variables, PC1 and PC2 will reflect these two clusters, with loadings from each cluster dominating each component.

— If variables are uncorrelated with each other, PCA will not provide much compression — each component will explain roughly the same proportion of variance, and the data may not be suitable for PCA.

Bartlett’s Test of Sphericity (shown in the correlation output): Tests the null hypothesis that the correlation matrix is an identity matrix (i.e., all variables are uncorrelated). A significant result (p < 0.05) confirms that the correlation matrix is not an identity matrix and that PCA is appropriate for the data.

Kaiser-Meyer-Olkin (KMO) Measure of Sampling Adequacy: Ranges from 0 to 1. Values above 0.6 indicate that the patterns of correlations are relatively compact and PCA should yield distinct and reliable components.

15.1 Interpretation from Figure 31

In our rice genotype dataset, GY and TGW show a strong positive correlation (r = 0.74, p < 0.01), indicating that high-yielding genotypes tend to also produce heavier grains. DTF shows a moderate negative correlation with GY (r = −0.52, p < 0.05), suggesting that later-flowering genotypes tend to yield less under these conditions. CL is positively correlated with DTF (r = 0.61, p < 0.01), reflecting the known association between late maturity and grain chalkiness. PH shows weak to moderate correlations with most traits. Bartlett’s Test of Sphericity is significant (p < 0.001) and the KMO value is 0.72, confirming that the correlation structure is suitable for PCA. The heatmap reveals two variable clusters: one comprising GY and TGW (positively associated), and another comprising DTF and CL (positively associated) — consistent with the loading patterns observed in PC1 and PC2.

TipA strong significant correlation between two variables does not mean one causes the other — it means they share common underlying variation. PCA exploits this shared variation to construct interpretable components. Always interpret correlation patterns in the context of your subject-matter knowledge.

Conversely, DTF and CL have negative loadings on PC1 — genotypes with low PC1 scores are less desirable for these variables in terms of yield optimization, but may be relevant if long duration or high chalkiness is the study objective. For PC2, PH has a strong positive loading. If plant height is your focus, the PC2-based index would be more relevant. It is recommended to use variables that are meaningfully correlated for PCA, as this helps construct a more reliable and interpretable index.

The biplot gives a visual representation of the relationships among variables and genotypes. Genotypes with high values for a specific variable are positioned in the direction of that variable’s arrow. The angle between variable arrows in the biplot indicates their correlation — smaller angles suggest high positive correlation, while angles close to 90 degrees suggest weak or no correlation, and angles close to 180 degrees suggest negative correlation.

In RAISINS, we calculate a scaled index score by converting the index score to a range of 0 to 1, making it easier to interpret and compare. This standardized approach ensures consistency in evaluating observations based on their index scores. To refine your selection, use the Select cutoff for Scaled Index Score feature shown in Figure 40, where you can choose the cutoff percentage to select observations above or below a threshold. The default cutoff is set at 75%. By toggling the up-arrow and down-arrow buttons below the cutoff selection, you can select the top or bottom percentage of observations as per your preference. Selected observations are highlighted in yellow in the table below, providing a clear visual cue. Additionally, a plot based on the index scores is displayed to aid in your analysis.

Combining all this information — MANOVA significance, PC eigenvalues, loadings, biplot patterns, HCPC cluster membership, correlation structure, and scaled index scores — the researcher can arrive at an overall conclusion that is statistically sound and contextually relevant to their study. Superior genotypes identified through PCA-based indices can be validated against individual ANOVA results and post-hoc groupings to ensure that their superiority is not confined to one or two traits but reflects genuine multivariate excellence.

19 Preparing data in MS Excel

Open a new blank sheet in MS Excel with only one sheet included, and avoid adding any unnecessary content. The dataset should follow a column-based format, where the first column represents the observation labels (e.g., genotype names, variety names, sample identifiers) — you can name this column appropriately, such as “Genotype” or “Treatment.” All variables under study (e.g., GY, PH, DTF, TGW, CL) should be arranged in separate columns, with one row per observation. The file can be saved in CSV, XLS, or XLSX format, but CSV is recommended as it is lighter and enables faster loading. Ensure that there are no unwanted spaces in column names or observation labels. For reference, see the structure shown in Figure 42. The data should be arranged as illustrated in Figure 2 and Figure 43.

Dataset Creation Rules

1. Column Naming Convention - No spaces allowed in column names.
- Use underscores (_) or full stops (.) for separation. - Avoid symbols and special characters like %, # etc. 2. Data Arrangement - Start data arrangement towards the upper-left corner.
- Ensure the row above the data is not blank. 3. Cell Management - Avoid typing or deleting in cells without data.
- If needed, select affected cells, right-click, and select Clear Contents. 4. Column Relevance - Name all columns meaningfully.
- Exclude unnecessary columns not required for analysis. 5. Observation Labels - Ensure each row has a unique observation label in the first column.
- Do not use numbers only as labels if they could be mistaken for data values.

How to Save as CSV in MS Excel

1. Open Your Workbook

-   Ensure your data is arranged properly with only one sheet.

  1. Click ‘File’ Menu

    • Go to the top-left corner and click on File.

  2. Choose ‘Save As’ or ‘Save a Copy’

    • Select the location where you want to save your file.

  3. Set File Type to CSV

    • In the ‘Save as type’ dropdown menu, choose CSV (Comma delimited) (*.csv).

  4. Name Your File

    • Enter a relevant file name without spaces (use underscores if needed).

  5. Click ‘Save’

    • Click Save to export the file.

💡 Tip: Before saving, double-check that your data is on the first sheet and follows the required format (e.g., no empty rows above the data, meaningful column names, one row per observation).

23 View data

View Data serves as the primary diagnostic tool for ensuring data integrity before analysis.
